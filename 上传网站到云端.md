这套流程是目前全球开发者最主流、最丝滑的**“持续集成与部署 (CI/CD)”**方案。它让你从一个本地的“文件夹”变成了一个全球可访问的“艺术品”。

以下是为你整理的**《从代码到全球上线：GitHub + Cloudflare Pages 存档手册》**：

---

### 📦 第一阶段：把代码交给 GitHub（建立云端仓库）

GitHub 在这里扮演的是“**代码中转站**”和“**版本管理器**”。

1.  **建立连接（一次性动作）**：
    *   在 GitHub 网页上新建一个 Repository（仓库）。
    *   在本地项目文件夹打开终端（PowerShell），输入：
        ```powershell
        git init                            # 初始化，让文件夹变身 Git 仓库
        git add .                           # 把所有代码“抓”起来（不包括沉重的 node_modules）
        git commit -m "First commit"        # 给这次代码打个包，起个名
        git branch -M main                  # 确认主分支叫 main
        git remote add origin [你的仓库链接]   # 告诉本地电脑：你的云端家地址在这
        git push -u origin main             # 正式把代码“推”上云端
        ```

2.  **核心原理（为什么工程很大但上传很快？）**：
    *   你的项目里有个 `.gitignore` 文件。它告诉 Git：**忽略 `node_modules` 文件夹**。
    *   GitHub 只存你的 **“菜谱”**（源代码），不存 **“原材料”**（依赖包）。这保证了仓库的轻量和纯净。

---

### 🚀 第二阶段：把 GitHub 接入 Cloudflare Pages（全球分发）

Cloudflare Pages 是你的“**云端加工厂**”和“**服务器**”。

1.  **关联仓库**：
    *   登录 Cloudflare，进入 **Workers & Pages** -> **Pages**。
    *   点击 **Connect to Git**，选中你刚才在 GitHub 建好的仓库。

2.  **配置“加工指令”（关键参数）**：
    *   **Framework preset (框架预设)**：选择 **None** (手动配置) 或 **Vite**。
    *   **Build command (构建命令)**：填入 `npm run build`。
        *   *作用：让 Cloudflare 在云端帮你把 React 代码压缩成极小的网页文件。*
    *   **Build output directory (输出目录)**：填入 `dist`。
        *   *作用：告诉 Cloudflare，打包好的成品都在这个文件夹里，把它发布出去。*

3.  **注入环境参数 (Environment Variables)**：
    *   设置 `NODE_VERSION = 20`。
    *   *作用：给云端加工厂升级到最新的引擎版本，确保支持 Vite 等新技术。*

4.  **自动化流程**：
    *   一旦点击 **Save and Deploy**，Cloudflare 会自动完成：**拉取代码 -> 安装依赖 -> 编译打包 -> 分布式部署**。

---

### 🔄 第三阶段：Vibecoding 的“无限循环”维护

这是最牛逼的地方：**你以后再也不用点 Cloudflare 网页了！**

当你想要更新网站（比如 OpenCode 帮你改了流体颜色）：
1.  在本地改好代码，测试 OK。
2.  在终端输入“三板斧”：
    ```powershell
    git add .
    git commit -m "update vibe"
    git push
    ```
3.  **奇迹发生**：Cloudflare 监控到 GitHub 代码变了，它会**自动**重新打包并更新你的 `xxxx.pages.dev` 网址。

### 🧐 你的疑问解答：

*   **部署到底在哪里？**
    你的网页并不是存在某一个固定的服务器上。Cloudflare 把你的 `dist` 文件夹同步到了它在全球 200 多个城市的 **CDN（内容分发网络）** 节点上。当你在北京访问，它就从北京的节点给你发数据；你在美国访问，它就从美国的节点发。
*   **为什么不需要 VPN 也能看？**
    因为 Cloudflare 在全球的网络权重极高，且 `.pages.dev` 目前在国内的连通性优于 `.vercel.app`。它通过全球加速技术绕过了很多网络阻塞。
*   **我可以删掉本地文件吗？**
    可以，但**千万别删 `.git` 文件夹和你的源码**。只要代码在 GitHub 上，你的网站就永远在线。如果你换了台电脑，只需要从 GitHub 把项目 `git clone` 下来，运行 `pnpm install`，就又能找回完整的开发环境。
