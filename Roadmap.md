## 一、数值平衡（Balance）

目前数值多为「凭感觉」设定，真实曲线需要遥测 + 试玩反馈。按重要程度：

### 1.1 分数曲线重标定
- **动机**：当前 Ante 目标 `300 / 500 / 900 / 1600 / 2800 / 5000 / 9000 / 16000` 增长曲线略平，中后期可能出现「早爆后躺平」或者「卡关死局」两极分化。
- **可度量目标**：
  - 新手 Ante 1 胜率 > 92%
  - Ante 8 胜率在 18%~28%
  - 每关的中位数分数应落在目标的 1.1x ~ 1.8x 区间
- **方案**：
  1. 在 `ScoreEngine` 里埋遥测钩子，记录每次 K-A 的单次总分、Combo 层数、完成时剩余步数
  2. 把目标公式从「硬编码数组」换成 `target(ante) = round(260 × 1.78^ante)` 这种可调参公式
  3. 提供 `?debug=curves` URL 参数以显示当前关卡内每一步的目标分数百分比
- **风险**：Ante 8 目标太低会破坏 Balatro 式的「最后一关都靠爆发」体验，不能简单线性压平。

### 1.2 Combo × X-Mult 上限保护
- **动机**：当前 Combo 是 `×1.5` 连乘，理论 7 层就 ×17x。如果再叠上「蜂后 X1.5」「关卡 X2」「最后一步 X3」等全局 X-Mult，单次 K-A 可能飙到 10 万+，把后续所有 Ante 一次打穿。这是 Balatro 式爽快，但需要设计成「极限构筑才能达到」而不是「随机散件也能到」。
- **方案**：
  - Combo 基数降到 `×1.4`，层 7+ 进入「衰减区」（每层再 +0.3 而非乘 ×1.4）
  - X-Mult 同源效果「多个取最大值」而不是连乘（例如两张昆虫卡都给 X1.5，只取 1.5 一次）
  - 预留一个昆虫卡「指数回归」（**Voider · 虚空织纹**）：X-Mult 改成加法而非乘法，作为「极限流」专属
- **风险**：过度削弱会让 Balatro 式极致爽感受损，必须在遥测上验证。

### 1.3 昆虫卡池 rarity 曲线
- **动机**：当前商店 `rollShopItems()` 是均匀随机，rarity 1~5 出现概率不分。
- **方案**：权重表
  ```
  rarity 1（Common）   : 45%
  rarity 2（Uncommon） : 30%
  rarity 3（Rare）     : 17%
  rarity 4（Epic）     : 6%
  rarity 5（Legendary）: 2%
  ```
  Ante 3 之后，Rare+ 的权重翻倍；Boss 战后固定掉一张 ≥3。
- **方案配套**：为 Legendary 设计独立「银框」视觉，增加取得仪式感。

### 1.4 经济曲线
- **动机**：目前 `INTEREST_CAP = 5`，与 Balatro 的 5 一致，但 clear bonus 只有 3 金。整体金币获取偏慢。
- **方案**：
  - 保留 interest cap，但提供一个 Uncommon 昆虫卡「**蝎 · 吝啬鬼**」：利息上限 +3，但每回合第一次移动 -20 Chips
  - Boss 战金币奖励从 15 提升到 25，并在商店里给「保留稀有道具」特权（花 5 金买下一关「锁」）
- **风险**：金币太多会让玩家一路堆稀有道具，滚雪球过强。

### 1.5 步数上限调整
- **动机**：25 步对熟手太松，对新手偏紧。
- **方案**：
  - 按 ante 动态（`stepLimit = 25 + floor(ante / 3)`），给后期更多腾挪空间
  - 或把步数改成「消耗品 · 时蜜」才能补的稀缺资源（当前 `time_honey +5 steps` 已经是这种思路），提高时蜜价值

### 1.6 强化牌（Enhancement）命中率与价值
- **动机**：现在每关开局随机 5 张牌强化，玩家常常感受不到强化（盖牌下面的强化翻到时已经不重要）。
- **方案**：
  1. 改成「优先分配给已翻开的牌」或「已翻开 70% 概率 + 30% 分给盖牌」
  2. 强化卡效果差异化重排：
     - `silk`：该张被消除时 +80 Chips（中等实用）
     - `venom`：消除后下一张操作 ×0.15 额外 Mult（高价值）
     - `gilded`：该张被移动时 +3 金（经济流）
     - `brittle`：该张消除后立刻翻一张邻列盖牌（探索流）
     - `dewdrop`：该张触发 K-A 时 +1 X-Mult（爆发流）
     - `petrified`：花色变为「任意」但 Chips 减半（破局流）

## 二、内容扩展（Content）

### 2.1 昆虫卡 10 → 25
- 新增方向：
  - **共振系（Resonance）**：与特定花色/颜色联动（例："每条红色 K-A 多给 ×0.3 X-Mult"）
  - **经济系（Economy）**：金币滚雪球（例："每关结束 +1 利息上限"）
  - **风险系（Risk）**：高收益但附加惩罚（例："Chips ×2，但每关开局强制弃掉 2 张牌"）
  - **演化系（Evolve）**：用次数达到阈值后升级（例："完成 3 次 Combo×5 后，本局剩余时间 Mult +10"）
- 目标数量：常用池 15 张 + 限定池 10 张（仅 Boss 战掉/商店 20% 概率出现）

### 2.2 卷轴 / 药水 扩充
- 当前有 5 卷轴 + 6 药水
- 建议扩到 10 / 12
- 新卷轴方向：
  - **毒雾卷轴**：给「整整一列」附上 venom
  - **缠丝卷轴**：把两张不同花色的牌「焊死」，视作同花色
- 新药水方向：
  - **重置之瓶**：把本关的「强化分布」重新随机（破解 enhancement 分配运气差的情况）
  - **时间倒流**：撤销上 1 步（但不退还 step）

### 2.3 Boss 战机制
- 当前 Boss 只是 +15 奖励 + UI 横幅，战斗规则没有变化
- 建议每个 Boss 有一条独特「禁令」：
  - **「结网者」**：K-A 消除只奖励一半 Chips（迫使玩家依赖 Combo）
  - **「食者」**：每回合第 3 步开始步数×2 消耗（压缩节奏）
  - **「暗眼」**：本关盖牌一律不显示翻开动画（考验记忆）
  - **「毒王」**：强化牌只有负面（brittle / petrified）
- 每个 Boss 对应一张限定 Legendary 昆虫卡作为通关奖励

### 2.4 关卡样板（Modifiers）
- Slay the Spire 式的房间变体，Ante 1~8 每关随机：
  - **骷髅关**：步数上限 -5，但分数目标 -20%
  - **宝库关**：每次移动 +1 金币
  - **雾霭关**：所有盖牌连翻 3 次才翻开
  - **遗迹关**：开局强化牌数量翻倍

### 2.5 角色（Spider 角色选择）
- 每个角色起手套装 + 一条被动 + 一条禁令，复刻 Slay the Spire 四职业模式
- 起手角色：
  - **织者 Weaver**：起手 `silk_scroll`，被动「同花色序列移动额外 +2 Chips/张」
  - **毒师 Venomist**：起手 `venom_scroll`，被动「venom 牌消除后 Mult ×1.2」
  - **金腹 Goldbelly**：起手 `gold_rain`，被动「利息上限 +2，但 Mult 来源 -1」
  - **虚影 Phantom**：开局多 1 张 face-up 牌，但所有金币奖励 -30%（硬核向）

## 三、元进度与 Roguelite 系统（Meta）

### 3.1 解锁树
- 首次通关解锁：新角色、新昆虫卡加入池
- 按完成度解锁，不要靠「刷币」
- 把已解锁 / 未解锁的「图鉴」做成一个前景丰富的页面（参考 Balatro 的 Collection）

### 3.2 挑战模式
- 每日种子（Daily Seed）：全球同一局牌，比谁分高。需要简单的排行榜后端（Supabase + RLS 足够）
- 周挑战：附加奇怪规则（"全程不使用消耗品"/"只用黑色花色" 等）

### 3.3 存档
- 关卡内自动存档（localStorage 即可）
- 跨端同步通过一个简单的 code 导入 / 导出（不需要账号）

### 3.4 统计
- 每局结束面板：最大 Combo、最高单次 K-A 分、使用的昆虫卡、剩余步数。打卡型数据让玩家有谈资。

## 四、UI / UX / 可读性（Polish）

### 4.1 Tutorial 与引导
- 当前只有 HelpModal，玩家第一次打开要自己点
- 建议：
  - 第一次开局时 pointer 指示「拖我」的动画
  - 第一次触发 Combo 时屏幕正中 tooltip 解释 Combo 机制
  - 第一次进入商店时，高亮「重掷（Reroll）」按钮

### 4.2 读数优化
- 把 `×1.5` 这种倍率字号做得和 `300` 主分数字号等大，玩家需要在一眼内看到乘数
- Combo 触发时把屏幕中央的数字做成全屏覆盖（短暂 300ms）

### 4.3 动效节奏
- 当前 K-A 动画 = 600ms 聚合 + 1.4s 裂纹 = 2s，连续 K-A 会让玩家觉得「等太久」
- 建议：第二次 K-A 开始动画节奏压缩到 60%，并跳过「聚合阶段」的细节

### 4.4 触屏 / 移动端
- 当前 canvas scale 是 `Math.min(sx, sy, 1)`，在手机上会变得很小
- 建议：
  - 竖屏布局（盘面 5 列 × 2 行拼接？或整体 90° 旋转）
  - 卡牌触屏可点选模式（点起始 → 点目标）替代拖拽
  - 抬起时给 haptic feedback（`navigator.vibrate(15)`）

### 4.5 色盲友好
- 当前红黑花色仅靠颜色区分；新增「花色形状徽记」开关（方片加菱形描边、红心加心形实心/空心等）

### 4.6 分辨率适配
- `GAME_WIDTH=1280` 硬编码，布局以 1920×1080 以上桌面观感最佳
- 应引入 `SafeArea` 容器 + 百分比布局：主牌盘 60%、HUD 15%、tray 15%、底部留 10%

## 五、美术与音乐（Art / Audio）

### 5.1 美术层级
目前全部由 Pixi.Graphics 程序化绘制，优点是体积小，缺点是手绘质感弱。按工作量从小到大：
1. **SVG 图标**：昆虫卡 glyph 换成 10 张 SVG，瞬间质感+30%
2. **卡面花色**：换成外部 `assets/suits/` 目录里的精美符号（红桃带光影等）
3. **Boss 立绘**：每个 Boss 一张 512×512 PNG（AI 生图即可）
4. **背景视差层**：两三层半透明背景，随鼠标 parallax 漂移
5. **全手绘卡面**：完整 52 张卡的手绘画面（成本高，放最后）

### 5.2 音乐
- 当前是纯 Web Audio 合成的 ambient drone + pad + 偶发钟琴。建议：
  - 把 BGM 换成 2 轨外挂（loop：~40s，boss：~50s），每轨 MP3 约 500KB
  - 保留程序合成作为「解锁模式」或「Lofi 模式」（玩家可切换）
- **动态音乐**：Combo ≥5 时自动混入「arpeggio 层」（已部分实现，可做成更激烈的版本）

### 5.3 SFX 多样化
- 目前每个事件只有一个合成音。建议每种事件准备 3 个变体，随机播放，避免重复听觉疲劳
- 重点提升：`playMove`（最常听到）、`playFlip`、`playClear`

### 5.4 粒子系统通用化
- 当前 `ScoreBurst`、`KACascade` 各自写粒子，建议抽象出一个 `ParticleEmitter` 工具类，支持：
  - 点源 / 线源 / 圆形发射
  - 粒子寿命 / 初速度分布 / 重力
  - 纹理（圆 / 星 / 自定义 Graphics）

## 六、可玩性深化（Feel / Depth）

### 6.1 连锁 K-A（Chain KA）
- 当 13 张 K-A 消除后，若盘面在掉落后还有「继发 K-A」，给 ×1.5 额外 bonus（类似宝石迷阵连锁）
- 实现：在 `handleMove` 的 cleared 分支里循环 `checkClearableSequence` 直到没有新序列

### 6.2 序列长度不对称奖励
- 当前 7+ 序列才能视作 K-A。可以让「4/5/6 张不完整序列」也能触发「Partial Clear」：
  - 4 张：+20 Chips、不消除
  - 5 张：+40 Chips、消除 +0.5 Mult
  - 6 张：+80 Chips、消除 +1.5 Mult、屏幕微震
  - 13 张：保持当前满奖励
- 让每一步都有小爽点

### 6.3 盖牌概率管理（Information Economy）
- Balatro 的爽感一部分来自「我看见下一张牌了」。Spider 的天然盖牌机制可以被强化成核心机制：
  - 一张「**水晶球**」消耗品：永久显示 3 张随机盖牌
  - 一条昆虫卡「**守望者**」：所有盖牌下角打小标记显示颜色（黑 / 红）
  - 让「信息差」成为可购买资源

### 6.4 Combo 复活机制
- 当前 Combo 一旦被任何异色移动打断就归零，太硬
- 新增昆虫卡「**续命蛛**」：每关允许 1 次 Combo 打断保留（防止一次失误全毁）

### 6.5 负面效果 / 诅咒
- Balatro 的「Voucher」里有「买一送一但总分-50%」这类博弈选项
- 为本作引入「诅咒」系统：Boss 通关时可选接受一个诅咒（例"本关每次移动 -3 Chips"），换一个小丑卡或 10 金

## 七、技术与工程（Engineering）

### 7.1 遥测
- 加入无后端的简单遥测（sendBeacon 到一个 Supabase Edge Function 或 Google Analytics 4 自定义事件）：
  - 开局 / 结局 / 每次 K-A / 胜负 / 商店购买
- 对数值调整是刚需

### 7.2 本地化
- 当前 i18n 只有 zh / en，已完善。下一步：
  - 加 ja（日本市场是 Roguelike 重要人群）
  - 把 UI 文案长度测试用 pseudo-locale（把字符串放大 40%）做过一遍

### 7.3 代码架构
- 当前 `Game.ts` 700 行，有点臃肿。建议拆分：
  - `GameEconomy.ts`（金币/商店/奖励）
  - `LevelController.ts`（关卡流转、胜负判定）
  - `InputController.ts`（拖拽/卷轴目标模式切换）
- 这样以后加新关卡类型、新模式会更轻松

### 7.4 种子 / 可重放
- 用一个 PRNG（seedrandom）替换所有 `Math.random`
- 每局开始生成一个 seed，允许「复制对局码」给朋友
- 做每日挑战和 Speedrun 社区的基础

### 7.5 性能
- 当前是每关重建所有 CardSprite。建议做一个 `CardSpritePool` 复用
- 粒子 `Graphics.circle()` 在高帧率下是瓶颈，可以换 `ParticleContainer` + `Sprite` 图集

### 7.6 构建与部署
- 体积优化：Pixi.js 里的 `RenderTargetSystem` / `WebGPURenderer` / `WebGLRenderer` 三个 chunk 合计 ~150 KB，通过 tree-shake 设置只留 WebGL 能省 ~40 KB
- 加一个 `base: './'` 的 vite 配置，可以部署到子路径（例如 GitHub Pages 的 `/card/`）
- PWA：加 service worker + offline 缓存，玩家可离线重开

## 八、社区与留存（Retention / Community）

### 8.1 录屏 / 分享
- 每局结束按钮「分享这局」→ 截图最终盘面 + 分数 + 昆虫卡组合 → 直接导出为 PNG
- 在 PNG 里嵌一个对局码（右下角二维码），朋友扫码即可在同一个种子上重放

### 8.2 Discord / 群社区
- 建一个官方 QQ/Discord 群，贴每日挑战的最高分榜
- 重要：人设「蛛网之主」可以以 Discord bot 的形式发表情图

### 8.3 Demo 版 / Steam 页面
- 当前 Web 版是天然的"试玩版"；浏览器端可加「演示版 · Ante 1-3」
- 完整版走 Steam / itch.io 一次性付费 $5-10（Balatro 的定价策略）
- Mobile App 单独做，按 pay-to-remove-ads 模式

## 九、风险与反模式清单（Anti-Patterns）

以下都是「看起来很想做但会毁游戏」的功能，放在这里作为**警告**：

### 9.1 ❌ 多人实时对战
- Spider Solitaire 的核心爽感是「思考 + 爆发」，加任何实时对手都会毁掉规划感
- 如果要社交，只做**异步**对局（每日种子、回放）

### 9.2 ❌ 局外付费购买道具 / 昆虫卡
- 付费越多碾压越严重 = Roguelike 结构崩溃
- 只做皮肤 / 卡背 / 音乐包等纯外观内购

### 9.3 ❌ 体力系统 / 广告换步数
- 一旦引入，Roguelike 玩家直接点差评
- 我们卖的是「爽感与构筑」，不是「时间差」

### 9.4 ❌ 太多全局 X-Mult 来源
- 连乘会让构筑坍塌到「只有一种能赢」。必须设定 X-Mult 最多 3 个同时生效 + 其余变加法

### 9.5 ❌ 过度细化 Tutorial
- Balatro 成功的原因之一是"你自己发现了 Combo"的 aha-moment
- Tutorial 只负责「拖卡/翻牌/达成目标」三件事，其他靠好奇心

### 9.6 ❌ 盲目抄 Balatro
- 我们的核心差异点在**空间管理**（10 列盘面 × 盖牌信息差），不是手牌管理
- 每次功能讨论都回到这个差异：**"这个改动会强化空间与信息差吗？"**；否则别加

4. **4.1 Tutorial 指针** + **4.4 移动端改造**：扩大玩家基础
5. **3.1 解锁树**：构建元留存
